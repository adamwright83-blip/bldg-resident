BOOKING PERSISTENCE TEST — SUCCESS

Root cause identified:
- Line 216-237 in Home.tsx: useEffect was blindly replacing `messages` state with `historyQuery.data` on every refetch (every 30 seconds)
- After booking, if the server returned stale/empty data (or the DB write hadn't completed), `setMessages([])` would trigger the empty state
- Empty state condition: `messages.length === 0` → shows centered logo + onboarding greeting

Fix applied:
- Only sync from server if local messages are empty OR server has more messages than local
- This preserves optimistic updates (booking confirmations) before they're persisted to DB
- Prevents wiping out the conversation after booking
