ROOT CAUSE: The user doesn't have a bldg_session cookie. The onboarding flow only works 
for authenticated users (those with a JWT cookie). The preview browser doesn't have the cookie.

SOLUTION: The onboarding flow needs to work even without authentication. Or we need to 
handle the case where there's no cookie by either:
a) Creating a guest session automatically
b) Making onboarding the mechanism that creates the user + sets the cookie

The current flow requires the user to come from Laundry Butler (which sets the JWT cookie).
For a "first-time user" experience, we need to handle the case where there's no cookie.

APPROACH: When getHistory returns user: null, the frontend should NOT auto-trigger onboarding.
Instead, it should show the greeting. When the user types anything, if there's no session,
we should create a guest user and set a cookie, THEN start onboarding.
